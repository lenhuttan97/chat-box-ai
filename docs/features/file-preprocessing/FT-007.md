# FT-007 — File Preprocessing Specification

## Overview

Xử lý file (images, documents) trên backend trước khi gửi đến AI provider. Dùng background job (Bull queue) để xử lý file, không lưu sau khi xử lý xong.

## Architecture

```
┌──────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   Frontend   │────►│  File Upload API │────►│  Save temp file │
└──────────────┘     │  POST /files     │     └────────┬────────┘
                     └──────────────────┘              │
                                                         ▼
┌──────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  AI Provider │◄────│  PreProcessor    │◄────│  Queue (Bull)   │
│  Gemini/Oll  │     │  Background Job  │     │  Process file   │
└──────────────┘     └──────────────────┘     └─────────────────┘
```

## User Requirements

1. **Upload file**: User có thể upload file kèm tin nhắn
2. **Background processing**: File được xử lý bất đồng bộ
3. **Multiple file types**: Hỗ trợ Images, PDF, DOCX, Excel
4. **Extract content**: Trích xuất nội dung text từ file
5. **Clean up**: Xóa file tạm sau khi xử lý

## Supported File Types

| Loại | Extension | Processing | Output |
|------|-----------|------------|--------|
| Images | jpg, jpeg, png, gif, webp | Resize + Base64 / Vision | Text description / Base64 |
| PDF | pdf | pdf-parse | Raw text |
| DOCX | docx | mammoth | Raw text |
| Excel | xlsx, xls | xlsx | Raw text |

## Database Schema

```prisma
model File {
  id            String   @id @default(uuid())
  filename      String
  originalName  String   @map("original_name")
  mimeType      String   @map("mime_type")
  size          Int
  status        String   @default("processing")  // processing, completed, failed
  extractedText String?  @map("extracted_text")
  error         String?
  createdAt     DateTime @default(now()) @map("created_at")
  updatedAt     DateTime @updatedAt @map("updated_at")

  @@map("files")
}
```

Fields:
- `filename`: Tên file trong storage
- `originalName`: Tên gốc khi upload
- `mimeType`: Loại file
- `size`: Kích thước (bytes)
- `status`: Trạng thái xử lý
- `extractedText`: Nội dung trích xuất


```
    │                                │
    │  POST /api/v1/files           │
    │  multipart/form-data          │
    │───────────────────────────────►│
    │                                │ Save temp file

    │                                │ Add to Bull queue
```

## File Extractors

### Image Extractor
```typescript
// Sử dụng Gemini Vision hoặc convert to base64
async extract(file: File): Promise<string> {
  // Option 1: Vision API (recommended for Gemini)
  const visionResult = await this.geminiProvider.analyzeImage(base64)
  
  // Option 2: Basic base64 (for Ollama)
  return base64Encode(buffer)
}
```

### PDF Extractor
```typescript
async extract(file: Buffer): Promise<string> {
  const pdf = await pdfParse(file)
  return pdf.text
}
```

### DOCX Extractor
```typescript
async extract(file: Buffer): Promise<string> {
  const result = await mammoth.extractRawText({ buffer: file })
  return result.value
}
```

### Excel Extractor
```typescript
async extract(file: Buffer): Promise<string> {
  const workbook = XLSX.read(file, { type: 'buffer' })
  // Convert all sheets to text
  return sheets.map(sheet => XLSX.utils.sheet_to_csv(sheet)).join('\n')
}
```

## Error Handling

| Error | Handling |
|-------|----------|
| File too large | Return 413, reject |
| Unsupported type | Return 400, reject |
| Extraction failed | Mark status=failed, return error |
| AI provider error | Retry 3 times, then fail |

## Acceptance Criteria

- [ ] User có thể upload file kèm tin nhắn
- [ ] File được xử lý bất đồng bộ (background job)
- [ ] Images: Trích xuất nội dung (Vision/Base64)
- [ ] PDF: Trích xuất text
- [ ] DOCX: Trích xuất text
- [ ] Excel: Trích xuất text từ all sheets
- [ ] File tạm được xóa sau khi xử lý
- [ ] Frontend có thể poll status để biết processing done
- [ ] Error handling cho các trường hợp fail
