# FT-009 — AI Service Refactor

## Overview

Refactor AI service layer để tách biệt business logic và provider-specific code, chuẩn bị cho multi-provider support.

## Mục tiêu

1. **Tách business logic**: AiService chỉ xử lý nghiệp vụ, không phụ thuộc provider cụ thể
2. **Factory pattern**: Dễ dàng thêm/switch providers (Gemini, Ollama, OpenAI)
3. **Error handling**: Validation, retry logic, structured logging

## Requirements

### 1. Refactor AiService

```typescript
// BEFORE: Mixed logic
async *sendMessage(options: SendMessageOptions) {
  const history = this.buildHistory(options)      // Business
  const stream = await this.geminiProvider.generateStream({...})  // Provider
}

// AFTER: Separated
async *sendMessage(options: SendMessageOptions) {
  const history = this.buildHistory(options)      // Business logic
  const provider = this.providerFactory.getProvider(options.provider)
  yield* provider.generate({ message, history, ... })  // Provider
}
```

### 2. Provider Interface

```typescript
interface AIProvider {
  name: string
  generate(options: GenerateOptions): AsyncGenerator<string>
}

class GeminiProvider implements AIProvider { ... }
class OllamaProvider implements AIProvider { ... }  // For FT-006
class OpenAIProvider implements AIProvider { ... }  // For FT-006
```

### 3. Error Handling

- Validation options (message length, temperature, tokens)
- Retry với exponential backoff
- Map errors ra appropriate HTTP responses
