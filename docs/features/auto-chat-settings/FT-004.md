# FT-004 — Auto Chat Settings Specification

## Overview

Tự động phân tích và bổ sung system prompt + context cho conversation sau tin nhắn đầu và định kỳ. **Bổ sung** chứ không thay thế cài đặt thủ công của người dùng (FT-003).

## User Requirements

1. **Tự động set sau tin nhắn đầu**: Sau khi user gửi tin nhắn đầu tiên, tự động phân tích và bổ sung system prompt + context
2. **Cập nhật định kỳ**: Sau mỗi 10 lượt tin nhắn, tự động phân tích lại và bổ sung
3. **Bổ sung, không thay thế**: Auto sẽ append thêm context vào systemPrompt của user (nếu có), không xóa giá trị user đã set
4. **Lưu cho mỗi conversation**: Mỗi conversation có settings riêng

## Logic

```
Tin nhắn mới gửi đến →
  Đếm số tin nhắn trong conversation →
    Nếu là tin nhắn thứ 2 (sau tin đầu) HOẶC số tin nhắn % 10 == 0 →
      Gọi AI phân tích context →
        systemPrompt = userSystemPrompt + "\n" + autoAnalysis
        contextToken = autoContextToken (nếu user chưa set)
```

## Database Schema (mở rộng)

```prisma
model conversation {
  id            String   @id @default(uuid())
  name          String
  user_id       String?  @map("user_id")
  system_prompt String?  @map("system_prompt")     // User set thủ công
  auto_prompt   String?  @map("auto_prompt")       // Auto bổ sung thêm
  context_token Int?     @default(4096) @map("context_token")
  temperature   Float?   @default(0.7) @map("temperature")
  max_tokens    Int?     @default(2048) @map("max_tokens")
  message_count Int      @default(0) @map("message_count")
  created_at    DateTime @default(now()) @map("created_at")
  updated_at    DateTime @updatedAt @map("updated_at")

  message       message[]

  @@map("conversation")
}
```

Fields:
- `name`: Tên conversation (user set)
- `system_prompt`: User set thủ công (FT-003)
- `auto_prompt`: Auto bổ sung thêm, không xóa user prompt
- `context_token`: Auto set nếu user chưa set
- `temperature`: User set thủ công
- `max_tokens`: User set thủ công
- `message_count`: Đếm tin nhắn để trigger auto update

## Integration Flow

1. **User gửi tin nhắn** → ChatService.processMessage()
2. **Kiểm tra message_count**:
   - message_count == 1 (tin thứ 2) HOẶC message_count % 10 == 0
3. **Gọi AI phân tích** → Lấy thêm context
4. **Merge**:
   - systemPrompt = (user system_prompt nếu có) + "\n" + (auto_prompt mới)
   - contextToken = user đã set ? giữ nguyên : auto set
   - temperature = user đã set ? giữ nguyên : auto set (default 0.7)
   - maxTokens = user đã set ? giữ nguyên : auto set (default 2048)
5. **Lưu vào DB** → conversation.auto_prompt, conversation.context_token

## AI Analysis Prompt

Khi cần phân tích để bổ sung context:

```
Phân tích đoạn chat sau và trả về JSON:
{
  "context补充": "Một hoặc hai câu bổ sung context cho system prompt (bổ sung thêm, không thay thế)",
  "contextToken": số token phù hợp (1024, 2048, 4096, 8192, 16384, 32000, 64000, 100000),
  "temperature": số từ 0.1 đến 1.0,
  "maxTokens": số token phù hợp (512, 1024, 2048, 4096, 8192)
}

Chỉ trả về JSON, không giải thích gì thêm.

Conversation history:
{messages}
```

## Acceptance Criteria

- [ ] Sau tin nhắn đầu, tự động phân tích và bổ sung context
- [ ] Sau 10 lượt, tự động cập nhật lại
- [ ] Bổ sung chứ không thay thế user settings (system_prompt + auto_prompt)
- [ ] contextToken chỉ auto set khi user chưa set
- [ ] temperature và maxTokens chỉ auto set khi user chưa set
- [ ] UI chỉ hiển thị user-settable fields (auto fields ẩn)
