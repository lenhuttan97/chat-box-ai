# FT-006 — AI Provider Selection Specification

## Overview

Thêm lựa chọn AI provider ngoài Google Gemini, hỗ trợ Ollama với nhiều models. User có thể chọn provider/model riêng cho từng conversation.

**Nguyên tắc:** Reuse API hiện có, chỉ mở rộng chứ không thay đổi interface.

## User Requirements

1. **Chọn AI Provider**: Gemini hoặc Ollama cho mỗi conversation
2. **Chọn Model**: Danh sách models được config trong env
3. **Cấu hình Ollama**: 
   - Local endpoint (localhost:11434)
   - Remote endpoint (Ollama hosted)
4. **Streaming Response**: Hỗ trợ streaming cho cả Gemini và Ollama

## Configuration (Environment)

```env
# Ollama Configuration
OLLAMA_ENABLED=true
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODELS=llama3,mistral,phi3,codellama,gemma

# Gemini Configuration (existing)
GEMINI_API_KEY=xxx
GEMINI_MODEL=gemini-1.5-flash
```

## Database Schema

```prisma
model conversation {
  id            String   @id @default(uuid())
  name          String
  user_id       String?  @map("user_id")
  device_id     String?  @map("device_id")
  
  // AI Provider Settings
  provider      String   @default("gemini") @map("provider")  // "gemini" | "ollama"
  model         String?  @map("model")                        // model name (llama3, mistral, etc.)
  
  // Existing settings
  system_prompt String?  @map("system_prompt")
  auto_prompt   String?  @map("auto_prompt")
  context_token Int?     @default(4096) @map("context_token")
  temperature   Float?   @default(0.7) @map("temperature")
  max_tokens    Int?     @default(2048) @map("max_tokens")
  
  created_at    DateTime @default(now()) @map("created_at")
  updated_at    DateTime @updatedAt @map("updated_at")

  message       message[]

  @@map("conversation")
}
```

Fields:
- `provider`: AI provider ("gemini" | "ollama"), default: "gemini"
- `model`: Model name (từ config), nullable

## Ollama API Integration

### Base URL
- Local: `http://localhost:11434`
- Remote: Configurable via env

### API Endpoints

#### List Models
```
GET /api/tags
```
Response:
```json
{
  "models": [
    { "name": "llama3:latest", "size": 3826793472 },
    { "name": "mistral:latest", "size": 4109854720 }
  ]
}
```

#### Chat Completion (Streaming)
```
POST /api/chat
Content-Type: application/json

{
  "model": "llama3",
  "messages": [
    { "role": "system", "content": "You are a helpful assistant." },
    { "role": "user", "content": "Hello" }
  ],
  "stream": true
}
```

Response (SSE):
```
data: {"message":{"role":"assistant","content":"Hello"},"done":false}
data: {"message":{"role":"assistant","content":" there"},"done":false}
data: {"done":true}
```

## Integration với Chat Service

### Existing API (reuse)
```
POST /conversation/messages  ← Dùng lại API hiện tại
PATCH /conversations/:id    ← Thêm provider/model fields
```

### Service Pattern (mở rộng)
```
AiService
├── GeminiProvider (existing)
└── OllamaProvider (new)
    ├── listModels()
    ├── chat(model, messages, stream)
    └── parseSSEResponse()
```

### Flow
```
User gửi tin nhắn → POST /conversation/messages
    │
    ▼
Lấy conversation (đã có provider, model)
    │
    ├── provider == "gemini" ──► GeminiProvider.chat()
    │
    └── provider == "ollama" ──► OllamaProvider.chat()
         │
         ▼
    Stream response về frontend (giữ nguyên SSE format)
```
