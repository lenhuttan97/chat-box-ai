# Skill: Phân tích Yêu cầu

## ⚠️ Tiên quyết
Phải chạy skills/clarify.md và nhận xác nhận từ user TRƯỚC.

---

## Quy trình

### Bước 1 — Phân loại yêu cầu
- Feature mới vs Cải tiến feature cũ
- Frontend only / Backend only / Full-stack
- Có ảnh hưởng DB schema không?
- Có ảnh hưởng AI integration không?

### Bước 2 — Viết User Stories
Format: "Là [persona], tôi muốn [action] để [outcome]"
Kèm Acceptance Criteria rõ ràng.

### Bước 3 — Xác định Technical Impact
- Prisma schema thay đổi gì?
- API endpoints nào cần tạo/sửa?
- Components React nào cần tạo/sửa?
- Redux state thay đổi gì?

### Bước 4 — Output
Điền vào template output-templates/analysis-report.md
